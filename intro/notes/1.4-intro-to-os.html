üìñ 1. Storytime: The Birth of Operating Systems

Before we dive into technical details, let me tell you a story. A story about rebellion, genius, and how a couple of stubborn programmers changed the world forever ‚Äî and in the process, absolutely smoked the billion-dollar corporations at their own game.

üèõÔ∏è The Dark Ages: Before Personal Computers

1960s-1970s

Imagine a world where computers filled entire rooms. Where you couldn't just "use" a computer ‚Äî you had to submit punch cards and wait hours (sometimes days) for your results. These were the mainframe days, and computing was reserved for universities, governments, and massive corporations.

In 1969, at AT&T's Bell Labs, something magical happened. Ken Thompson and Dennis Ritchie created Unix ‚Äî an operating system written in the C programming language (which Ritchie also invented). Unix was elegant, portable, and introduced concepts we still use today: hierarchical file systems, pipes, and the shell.

üí° Fun Fact: Unix was originally a play on words from "Multics" (another OS project). Where Multics tried to do everything, Unix did just a few things ‚Äî but did them incredibly well.

‚úä The Rebellion: Enter Richard Stallman (1983)

September 27, 1983

Richard Stallman, a programmer at MIT's AI Lab, was furious. Software was becoming proprietary. Companies locked away their code. You couldn't study it, modify it, or share it. For Stallman, this was a moral crisis.

So he did something radical: he announced the GNU Project. His mission? Create a complete, free (as in freedom) operating system.

"I consider that the Golden Rule requires that if I like a program I must share it with other people who like it."
‚Äî Richard Stallman, GNU Manifesto (1985)

GNU stood for "GNU's Not Unix" ‚Äî a recursive acronym (programmers love those). Stallman and his team created GCC (the compiler), Bash (the shell), Emacs (the text editor), and countless utilities. They built almost everything needed for a complete operating system.

Almost. They were missing one critical piece: the kernel.

üêß The Finnish Student Who Changed Everything (1991)

August 25, 1991

A 21-year-old computer science student in Finland named Linus Torvalds posted this message to a Usenet newsgroup:

"Hello everybody out there using minix ‚Äî I'm doing a (free) operating system (just a hobby, won't be big and professional like gnu) for 386(486) AT clones."
‚Äî Linus Torvalds

"Won't be big." Right.

Linus had created a kernel ‚Äî the missing piece of GNU. When combined with all the GNU tools Stallman's team had built, you had a complete, free operating system: GNU/Linux.

‚ö° The Impact Today:
‚Ä¢ Linux runs 96.3% of the top 1 million web servers
‚Ä¢ All of the world's top 500 supercomputers run Linux
‚Ä¢ Android (2.5 billion devices) is built on the Linux kernel
‚Ä¢ Most of the internet's infrastructure runs on Linux
‚Ä¢ NASA, SpaceX, and the ISS run Linux

Two guys ‚Äî one driven by moral conviction, one by curiosity ‚Äî created an operating system that now powers most of the world. FOR FREE. That's what I mean when I say they absolutely smoked the commercial giants.

ü™ü Meanwhile, in the Corporate World: Microsoft's Journey

1981

IBM needs an OS for their new personal computer. Bill Gates licenses MS-DOS to them ‚Äî crucially, he keeps the rights to sell it to other manufacturers. This decision makes Microsoft.

1985

Windows 1.0 launches ‚Äî a graphical shell running on top of DOS. It's clunky, but it's the beginning.

1995

Windows 95 changes everything. The Start menu, taskbar, and 32-bit architecture. Microsoft dominates the desktop.

2001

Windows XP merges the consumer (9x) and business (NT) lines. It becomes legendary for its stability and longevity.

2015 ‚Üí 2021

Windows 10 brings "Windows as a Service." Then Windows 11 arrives with a modern UI and native Android app support. Oh, and WSL ‚Äî Windows Subsystem for Linux. Microsoft finally embraced Linux.

üçé Apple's Transformation: From Mac OS to macOS

1984

The original Macintosh launches with the classic "Mac OS" ‚Äî a revolutionary GUI that influenced everything after it.

2001

Apple releases Mac OS X ‚Äî built on Darwin, a Unix-based kernel derived from BSD and Mach. Suddenly, Macs had the power of Unix under the hood.

2020

macOS Big Sur arrives alongside Apple's M1 chip ‚Äî ARM-based silicon that fundamentally changes the game for performance and efficiency.

üì± The Mobile Revolution: Android's Linux Heart

2003

Android Inc. is founded. Their secret weapon? Building on the Linux kernel.

2005

Google acquires Android. They see the future of mobile.

2008 ‚Üí Today

First Android phone launches. Today, Android powers 72% of all smartphones globally. That's over 2.5 billion active devices ‚Äî all running Linus's kernel.

üîê Why This Matters for Cybersecurity:

‚Ä¢ Open Source = Auditable Code ‚Äî You can verify what Linux is doing. With proprietary OS, you trust the vendor.
‚Ä¢ Understanding Lineage = Understanding Vulnerabilities ‚Äî Windows vulnerabilities differ from Unix ones.
‚Ä¢ Attack Surfaces Vary ‚Äî Servers (Linux) vs Endpoints (Windows) vs Mobile (Android/iOS)
‚Ä¢ 80% of enterprise breaches involve Windows endpoints ‚Äî know thy target.

üñ•Ô∏è 2. What is an Operating System?

Now that you know the history, let's understand what an operating system actually does. At its core, an OS is the layer between you and the hardware.

üë§ USER (You)
‚Üï
üì± APPLICATIONS (Browsers, Tools, Games)
‚Üï
‚öôÔ∏è OPERATING SYSTEM (Kernel + System Services)
‚Üï
üîß HARDWARE (CPU, RAM, Disk, Network)

The OS as a Translator

When you click "Save" in a document, you don't manually tell the hard drive to write specific bits to specific sectors. The operating system handles that translation.

Your application says: "Save this file"

The OS translates it to: "Write these bytes to sectors 34892-34910 on disk 0, update the file table, set permissions, record the timestamp..."

Kernel Space vs User Space

Kernel Space User Space
Full hardware access Limited, controlled access
Operating system core (kernel) Applications, user programs
Device drivers Browsers, games, tools
Crash here = system crash (BSOD, kernel panic) Crash here = app closes, system continues

System Calls: The Bridge

Applications in user space can't directly access hardware. They must request services through system calls ‚Äî the official API between user space and kernel space.

Common System Calls:
‚Ä¢ open(), read(), write(), close() ‚Äî File operations
‚Ä¢ fork(), exec(), exit() ‚Äî Process management
‚Ä¢ socket(), bind(), connect() ‚Äî Networking
‚Ä¢ mmap(), brk() ‚Äî Memory management

üîß 3. Virtualization & Modern Concepts

Modern computing isn't just about one OS on one machine anymore. Understanding these concepts is critical for cybersecurity work.

Concept What It Is Use Case
Bare Metal OS installed directly on physical hardware Daily driver, production servers, gaming
Virtualization OS running on virtual hardware (VMs) Labs, malware analysis, testing
Type 1 Hypervisor Runs directly on hardware (ESXi, Hyper-V) Enterprise servers, data centers
Type 2 Hypervisor Runs on host OS (VirtualBox, VMware Workstation) Home labs, development
WSL / WSL2 Linux kernel running inside Windows Best of both worlds, development
Containers Isolated processes sharing the host kernel (Docker) Deployment, microservices, CI/CD

Why Virtualization Matters for Security

üß™

üîÑ

‚öôÔ∏è 4. Core OS Functions (Practical Skills)

Every operating system handles these five core functions. As a security professional, you need to know how to inspect, manage, and manipulate each one across all major platforms.

4.1 Process Management

A process is a running instance of a program. Every application, service, and malware sample runs as a process. Knowing how to find, analyze, and kill processes is fundamental.

Viewing Processes

ü™ü Windows
# Task Manager (GUI)
taskmgr

# Command line - list all processes
tasklist

# PowerShell - detailed process info
Get-Process

# Find process by name
Get-Process -Name "chrome"

# Find by PID
Get-Process -Id 1234
üêß Linux
# List all processes
ps aux

# Interactive process viewer
top
htop  # Better version

# Find process by name
ps aux | grep nginx

# Process tree
pstree

# View /proc filesystem
ls /proc/[PID]/
üçé macOS
# Activity Monitor (GUI)
open -a "Activity Monitor"

# Command line
ps aux

# Top processes
top

# Find process
ps aux | grep Safari

Killing Processes

ü™ü Windows
# Kill by name
taskkill /IM notepad.exe

# Force kill by PID
taskkill /F /PID 1234

# Kill and all child processes
taskkill /F /T /PID 1234

# PowerShell
Stop-Process -Name "chrome" -Force
Stop-Process -Id 1234 -Force
üêß Linux
# Graceful terminate (SIGTERM)
kill 1234

# Force kill (SIGKILL)
kill -9 1234

# Kill by name
killall nginx
pkill nginx

# Kill all processes of user
pkill -u username
üçé macOS
# Same as Linux
kill 1234
kill -9 1234

# Kill by name
killall Safari

# Using pkill
pkill -f "process name"
üîê Security Relevance:
‚Ä¢ Malware often spawns multiple processes ‚Äî learn to identify and kill process trees
‚Ä¢ Look for unusual parent-child relationships (e.g., Word spawning PowerShell)
‚Ä¢ Check for processes with high privileges (SYSTEM, root) that shouldn't have them
‚Ä¢ Tools like Process Monitor (Windows) reveal what processes do

ü™ü 5. Windows Internals Deep Dive

Windows is the primary target for most attacks (80% of enterprise endpoints). Understanding its internals is non-negotiable for security professionals.

NTFS: The Windows File System

Master File Table (MFT)

The MFT is the heart of NTFS. It's a database containing entries for every file and directory ‚Äî including metadata, timestamps, and data locations.

MACB Timestamps:
‚Ä¢ Modified ‚Äî Content last changed
‚Ä¢ Accessed ‚Äî File last opened
‚Ä¢ Changed (MFT) ‚Äî Metadata last changed
‚Ä¢ Born ‚Äî File creation time

Attackers often try to modify these (timestomping) to hide activity!

Alternate Data Streams (ADS)

NTFS allows multiple data streams per file. Malware loves this for hiding payloads. 

# Create hidden stream
echo "hidden data" > normal.txt:hidden.txt

# View streams
dir /R
Get-Item .\normal.txt -Stream *

# Read hidden stream
Get-Content .\normal.txt -Stream hidden.txt

# Check all files in directory for streams
Get-ChildItem -Recurse | ForEach-Object { Get-Item $_.FullName -Stream * }

Windows Registry

The registry is a hierarchical database storing OS and application configuration. It's a goldmine for forensics and a playground for persistence.

Hive Content File Location
HKEY_LOCAL_MACHINE (HKLM) System-wide settings C:\Windows\System32\config\
HKEY_CURRENT_USER (HKCU) Current user settings C:\Users\{user}\NTUSER.DAT
HKEY_USERS (HKU) All user profiles Aggregated from user hives

Common Persistence Locations

# Run keys (execute at login)
HKCU\Software\Microsoft\Windows\CurrentVersion\Run
HKLM\Software\Microsoft\Windows\CurrentVersion\Run
HKLM\Software\Microsoft\Windows\CurrentVersion\RunOnce

# Services
HKLM\SYSTEM\CurrentControlSet\Services

# Scheduled Tasks
C:\Windows\System32\Tasks\

# Query via PowerShell
Get-ItemProperty "HKLM:\Software\Microsoft\Windows\CurrentVersion\Run"
Get-ItemProperty "HKCU:\Software\Microsoft\Windows\CurrentVersion\Run"

Windows Services

# List services
Get-Service
sc query

# Start/Stop service
Start-Service -Name "ServiceName"
Stop-Service -Name "ServiceName"
net start ServiceName
net stop ServiceName

# Service details
sc qc ServiceName

# Find service executable path
Get-WmiObject win32_service | Select Name, PathName | Where-Object { $_.PathName -like "*suspicious*" }

PowerShell for Security

# Get running processes with network connections
Get-NetTCPConnection | Where-Object { $_.State -eq "Established" } | 
  Select LocalAddress, LocalPort, RemoteAddress, RemotePort, 
         @{Name="Process";Expression={(Get-Process -Id $_.OwningProcess).ProcessName}}

# Find recently modified files
Get-ChildItem -Path C:\ -Recurse -File | 
  Where-Object { $_.LastWriteTime -gt (Get-Date).AddDays(-1) }

# Check startup programs
Get-CimInstance Win32_StartupCommand | Select Name, Command, Location

# Get logged-in users
query user
Get-WmiObject Win32_LoggedOnUser

# Network shares
net share
Get-SmbShare

Process Monitor is the ultimate tool for understanding what processes are doing in real-time: file access, registry changes, network activity.

Download: Sysinternals Process Monitor

üêß 6. Linux Internals Deep Dive

Linux powers most servers, cloud infrastructure, and security tools. Mastering it is essential.

File System Hierarchy

Directory Purpose Security Relevance
/etc System configuration User accounts, services, cron jobs
/var Variable data (logs, mail, spool) Logs for forensics
/home User home directories User data, ~/.ssh keys, history files
/tmp Temporary files Malware staging area, world-writable

Special Permission Bits

Bit Numeric Effect Security Impact
SUID 4000 Execute as file owner Privilege escalation if misconfigured
SGID 2000 Execute as group owner Similar risk, check group privileges
Sticky 1000 Only owner can delete files Important for /tmp 
# Find SUID binaries (privilege escalation goldmine)
find / -perm -4000 -type f 2>/dev/null

# Find SGID binaries
find / -perm -2000 -type f 2>/dev/null

# Find world-writable files
find / -perm -o+w -type f 2>/dev/null

# Find files with no owner
find / -nouser -type f 2>/dev/null

üçé 7. macOS Internals Deep Dive

macOS is Unix-based (Darwin kernel = BSD + Mach). Many Linux commands work, but there are unique aspects.

Darwin/XNU Kernel

macOS uses the XNU kernel (X is Not Unix), a hybrid of:

  • Mach ‚Äî Microkernel handling low-level tasks (IPC, memory)
  • BSD ‚Äî Unix layer providing POSIX compatibility, file systems, networking

launchd: The macOS Init System

Unlike Linux's systemd, macOS uses launchd for service management.

Location Purpose Runs As
~/Library/LaunchAgents/ Current user agents User
/Library/LaunchAgents/ All users agents User
/Library/LaunchDaemons/ System daemons Root
/System/Library/LaunchDaemons/ Apple system daemons Root 
# List launch services
launchctl list

# Load/Unload a service
launchctl load /Library/LaunchAgents/com.example.plist
launchctl unload /Library/LaunchAgents/com.example.plist

# Start/Stop
launchctl start com.example.service
launchctl stop com.example.service

# Check for suspicious items
ls -la ~/Library/LaunchAgents/
ls -la /Library/LaunchAgents/
ls -la /Library/LaunchDaemons/

macOS Security Features

Feature What It Does Bypass Implications
SIP (System Integrity Protection) Protects system files from modification, even by root Disabling requires recovery mode boot
Gatekeeper Blocks unsigned/untrusted apps Check app notarization status
TCC (Transparency, Consent, Control) Controls app access to camera, mic, files TCC.db manipulation is common malware target
XProtect Built-in malware signatures Basic AV, signature-based only 
# Check SIP status
csrutil status

# Gatekeeper status
spctl --status

# List TCC permissions
# Database at ~/Library/Application Support/com.apple.TCC/TCC.db

# Check code signing
codesign -dv --verbose=4 /path/to/app

# Verify notarization
spctl -a -v /path/to/app
