üìñ Introduction

What is SED?

SED stands for Stream EDitor ‚Äî a powerful, non-interactive text processing tool used to perform basic text transformations on an input stream (a file or input from a pipeline).

SED reads input line-by-line, applies specified operations (like find-replace, delete, insert), and outputs the result ‚Äî all without opening a text editor.

Why SED?

Reason Explanation
Speed Processes massive files line-by-line without loading entire file into memory
Automation Perfect for scripts ‚Äî no manual editing needed
Precision Regex-powered pattern matching for surgical edits
Pipelines Works seamlessly with other tools (grep, awk, cat, etc.)
Portability Available on virtually every Unix/Linux system

History ‚Äî Who Created SED?

  • Creator: Lee E. McMahon
  • When: 1973-1974
  • Where: Bell Labs (same place that gave us Unix, C, and awk)
  • Based on: The ed line editor (hence "stream ed-itor")

SED was one of the earliest tools to support regular expressions (regex), making it a pioneer in text processing.

Who Uses SED?

Role Use Case
SysAdmins Config file modifications, log parsing
DevOps Engineers CI/CD pipeline text transformations
Penetration Testers Parsing tool outputs, cleaning wordlists
Developers Batch code refactoring, data cleaning
Data Analysts CSV/text preprocessing

üìê General Syntax

sed [OPTIONS] 'COMMAND' [INPUT_FILE]

Or with pipeline:

cat file. txt | sed 'COMMAND'

Syntax Breakdown

Component Description
sed The command itself
[OPTIONS] Flags that modify behavior (-i, -n, -e, etc.)
'COMMAND' The operation to perform (in single quotes)
[INPUT_FILE] File to process (optional if using pipeline)

Command Structure

sed '[ADDRESS]OPERATION[/PATTERN/REPLACEMENT/[FLAGS]]'
Part Meaning
ADDRESS Which lines to operate on (line number, range, or pattern)
OPERATION What to do (s=substitute, d=delete, p=print, etc. )
PATTERN What to find (supports regex)
REPLACEMENT What to replace with
FLAGS Modifiers (g=global, i=case-insensitive, etc.)

üö© All Options & Flags

Command-Line Options

Flag Long Form Description
-n --quiet / --silent Suppress automatic printing; only print when explicitly requested with p
-e --expression=SCRIPT Add multiple commands in one sed call
-f --file=SCRIPT_FILE Read commands from a file
-i --in-place[=SUFFIX] Edit file in-place (modify original file)
-i. bak ‚Äî Edit in-place but create backup with .bak extension
-r / -E --regexp-extended Use Extended Regular Expressions (ERE)
-s --separate Treat each file as separate stream
-u --unbuffered Load minimal data and flush output buffers frequently
-z --null-data Separate lines by NUL character (not newline)
--help ‚Äî Display help message
--version ‚Äî Display version info
-i modifies the original file! Always test without -i first or use -i.bak for backup.

Substitution Flags (used with s command)

Flag Description
g Global ‚Äî Replace ALL occurrences in line (not just first)
p Print ‚Äî Print the modified line
i / I Case-insensitive matching
NUMBER Replace only the Nth occurrence (e.g., 2 = second match)
w FILE Write ‚Äî Write modified lines to FILE

üìö Core Concepts

1. Pattern Space vs Hold Space

Space Description
Pattern Space Temporary buffer where current line is held for processing
Hold Space Secondary buffer for storing data across lines (advanced)
Think of Pattern Space as your "workspace" and Hold Space as a "clipboard."

2. Addressing ‚Äî Targeting Specific Lines

Address Type Example Description
Line Number 5 Only line 5
Line Range 5,10 Lines 5 through 10
Start to End 5,$ Line 5 to last line ($ = last line)
Pattern /error/ Lines containing "error"
Pattern Range /start/,/end/ From line matching "start" to line matching "end"
Step 1~2 Every odd line (start at 1, step by 2)
Negation 5! All lines EXCEPT line 5

3. SED Commands (Operations)

Command Name Description
s Substitute Find and replace
d Delete Remove lines
p Print Output lines (use with -n)
a Append Add text AFTER current line
i Insert Add text BEFORE current line
c Change Replace entire line with new text
y Transform Character-by-character replacement (like tr)
q Quit Stop processing after current line
r Read Insert contents of another file
w Write Write pattern space to file
= Line Number Print current line number
n Next Skip to next line
N Append Next Append next line to pattern space
h Hold Copy pattern space to hold space
H Hold Append Append pattern space to hold space
g Get Copy hold space to pattern space
G Get Append Append hold space to pattern space
x Exchange Swap pattern and hold spaces

4. Regular Expressions in SED

Basic Regex (BRE) ‚Äî Default

Symbol Meaning
. Any single character
* Zero or more of previous character
^ Start of line
$ End of line
[ ] Character class
[^ ] Negated character class
\( \) Capture group (escaped parentheses)
\1, \2 Backreference to capture group
\{n\} Exactly n occurrences
\{n,m\} Between n and m occurrences

Extended Regex (ERE) ‚Äî Use with -E or -r

Symbol Meaning
+ One or more of previous character
? Zero or one of previous character
( ) Capture group (no escaping needed)
| OR operator
{n,m} Quantifier (no escaping needed)

5. Delimiters

The / is the default delimiter, but you can use ANY character: 

# Default
sed 's/old/new/'

# Using | as delimiter (useful when pattern contains /)
sed 's|/path/to/old|/path/to/new|'

# Using # as delimiter
sed 's#old#new#'

# Using @ as delimiter
sed 's@old@new@'
When working with file paths, use | or # to avoid escaping slashes.

6. Special Characters in Replacement

Character Meaning
& The entire matched pattern
\1 ‚Äî \9 Backreference to capture groups
\U Uppercase the replacement
\L Lowercase the replacement
\u Uppercase first character
\l Lowercase first character
\E End case conversion

‚öîÔ∏è SED vs AWK vs GREP

Comparison Table

Task GREP SED AWK
Search/filter lines ‚úÖ Best ‚úÖ Good ‚úÖ Good
Simple find/replace ‚ùå ‚úÖ Best ‚úÖ Good
Complex find/replace ‚ùå ‚úÖ Best ‚úÖ Good
Field/column extraction ‚ùå ‚ö†Ô∏è Possible ‚úÖ Best
Numeric comparisons ‚ùå ‚ùå ‚úÖ Best
Arithmetic operations ‚ùå ‚ùå ‚úÖ Best
Line insertion/deletion ‚ùå ‚úÖ Best ‚úÖ Good
In-place file editing ‚ùå ‚úÖ Best ‚ö†Ô∏è Limited
Speed on large files ‚úÖ Fastest ‚úÖ Fast ‚ö†Ô∏è Slower

When to Use What?

Scenario Best Tool
"Find all lines with error" grep
"Replace all foo with bar" sed
"Print 3rd column of CSV" awk
"Sum of all values in column" awk
"Delete lines 5-10" sed
"Find IPs in log file" grep -E
"Transform date format" sed
"Filter rows where col3 > 100" awk

Combined Power Examples

# Example 1: Find and count IT employees
grep ",IT," emp.csv | wc -l

# Example 2: Extract IT employee names
grep ",IT," emp.csv | sed -E 's/^[^,]*,([^,]*),.*$/\1/'

# Example 3: Average salary of IT department
grep ",IT," emp.csv | awk -F',' '{sum+=$5; count++} END {print sum/count}'

# Example 4: Pipeline magic - IT devs in Mumbai, sorted by salary
grep ",IT," emp.csv | grep "Mumbai" | awk -F',' '{print $5, $2}' | sort -nr
Remember: These tools work BEST together in pipelines. Master each, then combine them for ultimate text processing power!

üìã Quick Reference Cheat Sheet

üöÄ SED CHEAT SHEET

SUBSTITUTION

sed 's/old/new/'          # Replace first occurrence
sed 's/old/new/g'         # Replace all occurrences
sed 's/old/new/gi'        # Case-insensitive, all occurrences
sed 's/old/new/2'         # Replace 2nd occurrence only
sed 's/old/new/gp'        # Replace and print
sed 's|/old|/new|'        # Use | delimiter for paths

ADDRESSING

sed '5s/old/new/'         # Only on line 5
sed '5,10s/old/new/'      # Lines 5-10
sed '5,$s/old/new/'       # Line 5 to end
sed '/pattern/s/old/new/' # Lines matching pattern
sed '1~2s/old/new/'       # Every odd line
sed '0~2s/old/new/'       # Every even line

DELETION

sed '5d'                  # Delete line 5
sed '5,10d'               # Delete lines 5-10
sed '$d'                  # Delete last line
sed '/pattern/d'          # Delete matching lines
sed '/^$/d'               # Delete empty lines
sed '/^#/d'               # Delete comment lines

PRINTING

sed -n '5p'               # Print line 5 only
sed -n '5,10p'            # Print lines 5-10
sed -n '/pattern/p'       # Print matching lines
sed -n '/start/,/end/p'   # Print between patterns

INSERT / APPEND / CHANGE

sed '5i New line'         # Insert before line 5
sed '5a New line'         # Append after line 5
sed '5c New line'         # Replace line 5
sed '$a End of file'      # Append at end

IN-PLACE EDITING

sed -i 's/old/new/g' file      # Edit directly (Linux)
sed -i.bak 's/old/new/g' file  # Edit with backup
sed -i '' 's/old/new/g' file   # Edit directly (macOS)

MULTIPLE COMMANDS

sed 's/a/A/g; s/b/B/g'         # Semicolon separated
sed -e 's/a/A/' -e 's/b/B/'    # Multiple -e flags
sed -f script.sed file         # Commands from file

SPECIAL CHARACTERS

sed 's/. */[&]/'           # & = matched pattern
sed 's/\(word\)/[\1]/'    # \1 = backreference
sed 's/. */\U&/'           # Uppercase all
sed 's/. */\L&/'           # Lowercase all
sed 's/./\u&/'            # Capitalize first char

COMMON PATTERNS

sed 's/[[: space:]]*$//'   # Remove trailing whitespace
sed 's/^[[:space: ]]*//'   # Remove leading whitespace
sed 'G'                   # Double space file
sed '/^$/d'               # Remove blank lines
sed 's/,/\t/g'            # CSV to TSV
sed -n '1p'               # Print first line (head -1)
sed -n '$p'               # Print last line (tail -1)

üéì Key Takeaways

Remember These, Loki!

  1. SED = Stream Editor ‚Äî processes text line by line without loading entire file
  2. Non-destructive by default ‚Äî use -i for in-place editing (carefully! )
  3. Regex-powered ‚Äî master regular expressions to master sed
  4. Perfect for: find/replace, line manipulation, text transformation
  5. Not ideal for: numeric operations, complex logic (use AWK instead)
  6. Always test first ‚Äî run without -i before modifying files
  7. Pipeline friendly ‚Äî combines with grep, awk, sort, etc.
  8. Delimiters are flexible ‚Äî use | or # when dealing with paths
  9. Backup everything ‚Äî use -i.bak when editing in place
  10. Practice daily ‚Äî the more you use it, the more natural it becomes
